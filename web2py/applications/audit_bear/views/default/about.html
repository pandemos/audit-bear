About Audit-Bear

Overview

AuditBear automates the analysis of log files produced by ES&S iVotronic voting machines. It is intended for election officials who use iVotronic machines and who would like to gain visibility into the elections they administer, as well as for election observers, candidates, and interested citizens who would like to analyze elections conducted on iVotronic voting machines.

The AuditBear web application analyzes the log files you upload and produces reports about various aspects of the election. For instance, it can detect some kinds of procedural errors that might cause votes to be lost. Also, it attempts to identify polling locations that were forced to stay open late or may have experienced long lines during election day. And, it can detect machines that experienced certain anomalous conditions or whose date or time was set incorrectly.

Collecting the needed files

To use the AuditBear web application, you will need to collect and upload log files produced by the ES&S software. In particular, you will need:

1. The EL 152 (Votronic Event Log) file.
The EL 152 file contains event log entries from all of the iVotronic voting machines used in the election.
2. The EL 155 (iVotronic Vote Images) file.
The EL 155 file contains vote images (also known as cast vote records) cast on the iVotronic voting machines. Each vote image records all of the votes placed by a single voter.
3. Optionally, the EL 68A (System Log) file.
The EL 68A file contains log messages from the ES&S central software, which list actions performed on the central election system. The EL 68A file is not required to use our software, but if you have it, we will be able to provide more useful and detailed reports. (Caution: The EL 68 is a different report and is not a suitable replacement for the EL 68A. Our software is designed to analyze the EL 68A, not the EL 68.)
Election officials with access to the ES&S Unity Election Reporting Manager (ERM) software can export these files from the ERM software after the election. Others will need to request these files from their local election official.

For more information on how to export these files from the ERM software, see the ERM User's Manual. It describes how to generate log reports, particularly the system log, event log, and vote image log.

When you have collected the files listed above, create a zip file containing all of the above files. Then, upload the zip file into the file upload form on the AuditBear main page.

Want to try it out?

Want to try out the AuditBear web application, but you don't have the necessary log files? No problem!

To see the AuditBear application in action, simply grab log files from the South Carolina Voting Information web site. They have helpfully collected log files from several South Carolina counties in the November 2010 and June 2010 elections. Download one of their zip files, upload it into the file upload form on the AuditBear main page, and you can see an example of a report produced by the AuditBear software.

Software

AuditBear is completely free to use.

In addition, the source code for our software is freely available on Google Code, under an open source license.
